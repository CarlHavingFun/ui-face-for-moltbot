# Changelog

## [Unreleased]

### 语音输入

- **语音输入**：识别完成后不再自动发送，文本填入输入框，由用户点击发送。
- **持续监听**：与语音输入同级，改为独立按钮（原为勾选项）；说唤醒词后说完、静默约 1.8 秒再发送一次；再次点击结束；约 30 秒无唤醒会清空当前内容并重新听。
- 持续监听防重复发送：记录刚发送内容与时间，4 秒内相同内容不再设静默定时器，3 秒内相同内容不再发送，避免晚到的 final 导致同一句发两次。
- 持续监听无法停止：修复点击「持续监听」停止时仍会重启的 bug（先移除按钮 `recording` 再 `recognition.stop()`）。

### UI

- 语音 / 持续监听按钮增加可见标签（「语音」「持续监听」）和悬停 title 提示。
- 持续监听按钮文案为「持续监听」。

### 兼容

- **iOS**：在 iPhone/iPad 上点击语音输入或持续监听时，不再先调用 `getUserMedia`，直接 `recognition.start()`，避免与麦克风抢占导致闪退。
